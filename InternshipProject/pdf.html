Overview

Animation-Arcade is a frontend project built to demonstrate how a modern, production-ready frontend workflow works in real-world teams.

The focus of this project is not only UI or animations, but automation — ensuring code quality, consistency, and reliability from development to deployment.

Automation Goals

  • Enforce JavaScript and CSS code quality
  • Automatically run tests on file changes
  • Block bad commits using Git hooks
  • Enforce clean and meaningful commit messages
  • Ensure CI checks match local checks

Step 1: Environment Consistency (Node.js)

Use NVM and lock the Node.js version across the entire project.

Implementation

.nvmrc
24.11.1

Result

Local development, Git hooks, and GitHub Actions all use the same Node version.

Step 2: Project Scripts & Tooling

All automation starts from package.json. Developers interact with the workflow using simple npm commands.

Key Scripts

{
  "scripts": {
    "lint": "eslint .",
    "lint:css": "stylelint \"css/**/*.css\"",
    "test": "node --experimental-vm-modules node_modules/jest/bin/jest.js --passWithNoTests",
    "test:watch": "nodemon --watch js --watch __tests__ --ext js --exec \"npm test\"",
    "prepare": "husky"
  }
}

These scripts ensure linting, testing, and Git hooks work consistently for every developer.

Step 3: JavaScript Linting (ESLint)

ESLint is used to catch bugs early and enforce modern JavaScript best practices.

ESLint Configuration (Flat Config)

import js from "@eslint/js";
import globals from "globals";

export default [
  js.configs.recommended,
  {
    languageOptions: {
      globals: globals.browser
    },
    rules: {
      "no-var": "error",
      "prefer-const": "error",
      "eqeqeq": ["error", "always"],
      "no-debugger": "error",
      "no-console": "warn"
    }
  }
];

Step 4: CSS Linting (Stylelint)

Stylelint ensures CSS remains valid, consistent, and easy to maintain.

Stylelint Configuration

{
  "extends": ["stylelint-config-standard"],
  "rules": {
    "block-no-empty": true,
    "color-no-invalid-hex": true,
    "max-nesting-depth": 3,
    "selector-max-id": 0,
    "no-duplicate-selectors": true
  }
}

Step 5: Unit Testing (Jest)

Jest is used to validate JavaScript logic automatically. Tests run locally, during commits, and again in CI.

Sample Function & Test

export function add(a, b) {
  return a + b;
}

test("adds two numbers", () => {
  expect(add(2, 3)).toBe(5);
});

This ensures broken logic never reaches the main branch.

Step 6: Commit Message Enforcement

Commitlint enforces the Conventional Commits standard to maintain a clean and readable Git history. 

❌ wrong message
✅ feat(animation): add bounce effect

Step 7: Pre-Commit Automation (Husky)

Husky runs automated checks before every commit. If any check fails, the commit is blocked.

Pre-Commit Hook

npx eslint .
npx stylelint "**/*.css"
npm test

Step 8: Continuous Integration (GitHub Actions)

GitHub Actions runs the same checks as local development, ensuring consistency across environments.

CI Workflow

name: Lint & Test

on: [push, pull_request]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version-file: .nvmrc
      - run: npm ci
      - run: npm test

Pull requests cannot be merged unless all checks pass.
